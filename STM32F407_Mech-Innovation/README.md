# STM32F407 三层洗菜系统第三代控制逻辑修改说明

本次修改围绕第三代机械结构重新整理控制逻辑和外设链路，重点是：电机通信统一走 CAN2，移除旧 K230/USART2/USART6 残留，恢复 OLED 与 CubeMX 外设配置一致，并将第三层从“下层传送带”改为“称重板 + 双 HX711 + 5/6 号丝杆推板 + 7 号打包电机”。

## 本次主要修改

### 1. 电机通信链路统一为 CAN2

- 电机控制链路固定为：

```text
CAN2 -> Emm_V5 -> can_SendCmd
```

- CAN2 使用 `PB12 = CAN2_RX`、`PB13 = CAN2_TX`。
- `main.c` 保留 CAN2 初始化、滤波器配置、启动和接收中断通知。
- 电机控制不再走 USART6。
- 旧 USART6 电机通信相关逻辑已清理。

### 2. 第三代电机编号与控制方式

| 电机编号 | 用途 | 控制方式 | 说明 |
| --- | --- | --- | --- |
| 1 | 上层传送带电机 1 | 速度模式 | 与 2 号同步 |
| 2 | 上层传送带电机 2 | 速度模式 | 与 1 号同步 |
| 3 | 中层传送带电机 1 | 速度模式 | 与 4 号同步 |
| 4 | 中层传送带电机 2 | 速度模式 | 与 3 号同步 |
| 5 | 第三层丝杆推板电机 1 | 位置模式 | 与 6 号同步 |
| 6 | 第三层丝杆推板电机 2 | 位置模式 | 与 5 号同步 |
| 7 | 旋转打包电机 | 速度模式 | 推板推动时同步启动，到位后继续运行 |
| 8 | 第三代不使用 | 不调度 | 已从自动流程移除 |

5/6 号丝杆推板参数：

- 推动方向：`CCW`
- 返回方向：`CW`
- 速度：`30 RPM`
- 加速度：`0`
- 位置模式目标脉冲：`320000`
- 当前版本暂不解析 CAN 到位反馈，按时间估算完成。

### 3. 第三代自动流程状态机

当前自动流程为：

```text
READY
-> UPPER_RUN
-> MIDDLE_RUN
-> MIDDLE_CUTTING
-> LOWER_WEIGH
-> LOWER_PUSH_AND_PACK
-> PACK_TAIL
-> LOWER_RETURN
-> WAIT_NEXT_CYCLE
-> UPPER_RUN
```

主要行为：

- 上层 1/2 号电机运行，水泵打开，到时停止。
- 中层 3/4 号电机运行，下料舵机打开，随后切刀动作。
- 第三层进入称重，读取左右两路 HX711。
- 称重完成后，5/6 号丝杆推板和 7 号打包电机同步启动。
- 推板到位时间到达后停止 5/6 号，7 号继续运行 `3000 ms`。
- 7 号停止后，5/6 号返回初始位置。
- 等待短延时后进入下一轮。

### 4. 双 HX711 称重

第三层称重板使用两路 HX711：

| 通道 | SCK | DT | 说明 |
| --- | --- | --- | --- |
| 左侧 HX711 | PB10 | PB11 | 左侧称重 |
| 右侧 HX711 | PB14 | PB15 | 右侧称重 |

称重结果按两路合成：

```text
weight_total = weight_left + weight_right
```

如果某一路未接入或超时，该路重量按 0 处理，避免阻塞主循环。

### 5. 蓝牙链路改为 UART4

- 蓝牙模块使用 `UART4`。
- 引脚：
  - `PA0 = UART4_TX`
  - `PC11 = UART4_RX`
- 波特率：`115200`
- 主循环每 `50 ms` 调用一次 `Process_Bluetooth_Command()`。

当前蓝牙命令：

| 命令 | 功能 |
| --- | --- |
| `@011\r\n` | 启动自动流程 |
| `@022\r\n` | 故障恢复/复位 |
| `@033\r\n` | 优雅停机 |
| `@044\r\n` | 急停 |

### 6. OLED 显示恢复为软件 I2C

OLED 不是串口屏，也不是硬件 I2C；当前使用软件 I2C：

| 信号 | 引脚 |
| --- | --- |
| OLED_SCL | PB8 |
| OLED_SDA | PB9 |

OLED 默认地址按 `0x3C`，代码中写地址为 `0x78`。

### 7. CubeMX 外设对齐

当前 CubeMX 主要外设配置：

| 外设 | 引脚 | 用途 |
| --- | --- | --- |
| CAN2 | PB12/PB13 | Emm_V5 电机 CAN 通信 |
| UART4 | PA0/PC11 | 蓝牙模块 |
| USART1 | PA9/PA10 | Debug 调试串口 |
| ADC1_IN1 | PA1 | 水位/触水检测 |
| OLED 软件 I2C | PB8/PB9 | OLED 显示 |
| HX711 左侧 | PB10/PB11 | 左侧称重 |
| HX711 右侧 | PB14/PB15 | 右侧称重 |
| TIM1_CH1/CH2 | PE9/PE11 | 下层舵机 |

### 8. 已移除或清理的旧逻辑

- 移除 K230 通信源码和业务引用。
- 移除 UART4 中 K230 接收分支。
- 移除 USART2/huart2 蓝牙旧链路。
- 移除 USART6 电机通信路径。
- 移除旧“下层传送带”自动流程调度。
- 移除 8 号电机自动流程调度。
- 移除中层舵机 CH3/CH4 控制逻辑。
- 清理旧串口命令帧变量 `rxCmd/rxCount/CMD_LEN`。

## 优雅停机与急停

### 优雅停机

收到 STOP 后按顺序停机：

1. 停止上层 1/2 号电机，停止进料。
2. 等待 `stopDelayMs = 5000 ms`。
3. 停止中层 3/4 号电机，关闭切刀和下料舵机。
4. 再等待 `stopDelayMs = 5000 ms`。
5. 停止 5/6 号丝杆推板和 7 号打包电机。
6. 关闭水泵、切刀、舵机等执行机构。
7. 回到 `MODE_IDLE`。

### 急停

急停会立即：

- 广播停止 CAN 总线电机。
- 广播失能电机。
- 关闭水泵、切刀和舵机。
- 系统进入 `MODE_ERROR`。

## 编译与验证说明

本工程使用 Keil MDK 手动编译。

建议执行：

1. Keil Clean。
2. Rebuild。
3. 确认无未定义符号。
4. 烧录后先单独验证外设，再验证自动流程。

上板验证重点：

- OLED 上电是否正常显示。
- UART4 蓝牙命令是否能启动/停止流程。
- CAN2 是否能驱动单个电机。
- 1/2、3/4、5/6 是否按同步逻辑运行。
- 5/6 号 `320000` 脉冲是否对应实际 300 mm 行程。
- 双 HX711 是否能分别读数并合成总重量。
- STOP 是否按优雅停机顺序执行。
- 急停是否能立即关闭所有执行机构。

## 当前已知注意事项

- 5/6 号丝杆推板当前按时间估算到位，没有使用限位开关，也没有解析 CAN 到位反馈。
- 如果总线上仍接有 8 号电机，急停广播命令可能会影响它，但软件自动流程不再维护 8 号状态。
- Keil 旧编译产物中可能仍残留历史对象文件名，执行 Clean/Rebuild 后会刷新。
- 继电器高低电平逻辑需要按实际模块确认。
